Explicacion general: always on y file-based

Las instrucciones personalizadas permiten guiar el comportamiento de Copilot mas alla de un prompt puntual. Puedes combinarlas para mantener consistencia tecnica y de comunicacion.

Always on

Son directivas persistentes que Copilot tiene presentes de forma continua en las respuestas del chat y en el trabajo del agente.

File-based

Son instrucciones escritas en archivos dentro del repo. Sirven para capturar normas del proyecto y aplicarlas cuando Copilot trabaja sobre ese codigo.

Crear instrucciones a nivel de workspace en VS Code

Usa directivas especificas para un proyecto concreto. Ideal para arquitectura, convenciones del equipo y requisitos tecnicos de ese repositorio.

  1. Abre el proyecto en VS Code.
  2. Ve a la configuracion de Copilot Chat para el workspace.
  3. Define instrucciones centradas en este repo: estilo, validaciones y limites de cambio.
  4. Guarda y prueba con una tarea real para validar que Copilot sigue esas directivas.

Crear instrucciones de perfil de usuario en VS Code Copilot

Son reglas globales para tu cuenta y se aplican en todos tus proyectos. Utilizalas para preferencias personales de comunicacion, idioma y formato de respuesta.

  1. Abre los ajustes de perfil de Copilot en VS Code.
  2. Escribe directivas transversales que quieras mantener siempre.
  3. Evita meter detalles de un solo repo para no contaminar otros contextos.
  4. Revisa periodicamente estas reglas para mantenerlas utiles y actuales.
